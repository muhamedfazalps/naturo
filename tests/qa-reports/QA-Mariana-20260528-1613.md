## Phase 4 — First-time User findings

I deliberately did not pre-load context, ran `naturo` cold, then walked the surface a first-time user would touch in their first ~10 minutes: `naturo`, `naturo --version`, `naturo --help`, then English-verb guesses for "I want to launch / screenshot / use AI / tap on something".

**1. Typo suggester returns semantically wrong matches for short queries (#889, NEW, P2)**

```text
$ naturo ai
Error: No such command 'ai'. Did you mean 'wait'?
                                           # zero semantic overlap; user expected an AI feature

$ naturo tap
Error: No such command 'tap'. Did you mean 'app'?
                                           # nearest semantic match is 'click', not 'app'
```

Compare with what *does* land:

```text
$ naturo apps     → 'app'      ✓
$ naturo clik     → 'click'    ✓
$ naturo zzqqxx   → (no suggestion)  ✓
```

Root cause: `difflib.get_close_matches` with default cutoff 0.6 is too lax for short (≤3 char) queries against naturo's 30+ top-level command set. Random character overlap (`'a'+'i' ∈ 'wait'`, `'a'+'p' ∈ 'app'`) pushes unrelated commands above the bar.

Suggested fix in #889 body: bump cutoff to 0.7, or extra-strict ratio check (≥0.85) when query length ≤3. Alternative: small semantic alias table (`ai`→`find --ai` hint, `tap`→`click` hint, `screenshot`→`capture`).

Distinct from #880 (missing subgroup suggestion) and #867 (hidden-command leak) — different bug, same overall surface (typo suggester needs hardening). All three fire within the first 60 seconds of `pip install naturo`.

**3. Hidden-command leak applies to `hotkey` as well as `snapshot` (comment on #867)**

```text
$ naturo key
Error: No such command 'key'. Did you mean 'hotkey'?

$ naturo --help | grep -E '^\s+hotkey\b'
(empty)                                              # hotkey is NOT in --help

$ naturo hotkey --help
  Press a hotkey combination (alias for 'press').
  Deprecated: use 'naturo press ctrl+c' instead of 'naturo hotkey ctrl+c'.
```

Same shape as the `snap → snapshot` leak that opened #867. Whichever predicate filters `snapshot` out of `--help` must also gate the typo suggester. Fix scope is one change covering both `snapshot` and `hotkey` (and any future hidden command). TC-0055 updated.

**4. First-time UX observations not worth filing individually**

- `naturo --version -j` outputs plain text ("naturo, version 0.3.1") — already #874. The `--version` output also doesn't reveal which extras are installed; a user who `pip install naturo` (without `[windows]`) has no obvious way to verify their install is complete.

- `naturo desktop list` still prompts `Install it now? [Y/n]:` on stdout (line 1) before the JSON envelope (line 4) when `pyvda` is missing — already #869. Confirmed unchanged on `70e6591`.
