## Phase 4 — AI Agent Builder simulation

Built a 32-msg MCP client (`.work/qa-r137/mcp_probe2.py`). Findings:

### NEW BUG #890 — MCP `list_snapshots` fails 100% of calls (P1)

Wrapper at `naturo/mcp/_snapshot.py:165` passes `limit=limit` to `SnapshotManager.list_snapshots()` (defined at `naturo/snapshot.py:632`) which takes **no `limit` parameter**. Every MCP call (default args or with explicit `limit`) returns:

```json
{"success": false, "error": {"code": "INTERNAL_ERROR",
  "message": "TypeError: SnapshotManager.list_snapshots() got an unexpected keyword argument 'limit'"}}
```

CLI `naturo snapshot list -j` works correctly — bug is purely in the MCP wrapper. For an AI agent enumerating snapshots through MCP, this capability is **100% unavailable**. Filed: **#890** (v0.3.4, P1).

### NEW BUG #891 — MCP tools silently drop unknown arguments (P2)

Confirmed via `focus_window`. Schema declares `title`; a plausible typo `window_title` is **silently dropped** by the Pydantic-generated `*Arguments` model (Pydantic v2 default is `extra='ignore'`). Effect:

- `focus_window {"title":"claude"}` → success
- `focus_window {"window_title":"claude"}` → falls back to default `title=None` → resolves to "foreground" → `WINDOW_NOT_FOUND "Window not found: foreground"` (no diagnostic that `window_title` was ignored)
- `focus_window {"title":"claude","bogus":"x"}` → success, `bogus` silently dropped

For an agent calling a tool whose schema has drifted between naturo versions — or just typing a slightly different param name — the calls succeed in unexpected ways with no isError signal. Filed: **#891** (v0.3.4, P2).

### EXTENSION to #881 — NaturoCoreError leak surface is 8 tools, not 3

Original issue cites type_text/click/press_key. R137 probe confirms the leak across **8 MCP tools** in NO_DESKTOP_SESSION. Two of them additionally leak the raw Python `repr()` of the argument inside the error message:

- `press_key {key:'F1'}` → `NaturoCoreError: key_press('F1'): System/COM error`
- `hotkey {keys:['ctrl','c']}` → `NaturoCoreError: key_hotkey(('ctrl', 'c')): System/COM error`

Also confirmed: `TypeError:` Python class name leaks too (via #890). The mapping layer must hide **all** internal exception class names, not just `NaturoCoreError`. Comment posted on #881 with full 8-tool matrix and a verification-gate proposal.

## Top 3 Risks

1. **MCP error-surface is the agent product's weakest layer.** Of the four MCP families I probed (`list_*`, `capture_*`, write-ops, lookup), **three** have systemic bugs visible to a first-touch agent author: silent-success in NO_DESKTOP (#883/#885), exception class names leaking in error messages (#881/#890), and unknown-arg silent drops (#891). An AI agent author evaluating naturo via MCP hits at least one of these within the first 10 tool calls. This is qualitatively worse than the CLI surface today, and naturo's marketing leads with "AI Agent Ready" (per #887/TC-0075). The MCP layer needs a hardening pass before any AI-agent demo.

2. **`list_snapshots` end-to-end break (#890) is a regression caught only because QA happened to probe the snapshot surface.** The wrapper-vs-manager signature mismatch must have been introduced by a refactor that touched one side but not the other. No CI test exists for MCP tool wrappers calling the manager API — the entire `naturo/mcp/_*.py` surface should be covered by import-time + smoke-call tests. This is the second time in two months a Python-only refactor broke an interface (cf. #870 — Windows-only test failures uncaught). Suggest a CI step that imports each MCP tool module and runs it against a mocked desktop fixture.

3. **The 5 SendInput-blocked status:done remain unverified (#786, #788, #807, #840, #843).** Restructured ship gate (per Orc R134) decouples #863 from v0.3.2 — these 5 need verification from a **console-session QA invocation**, which still has not happened ~24h after the restructure. If the ship gate is to hold, someone (Ace or an agent running from a real console) needs to drive that verification before #885 closes. Otherwise we'll have #885 fixed but the ship still blocked on these 5.
