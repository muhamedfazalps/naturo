## Phase 4 — Chinese User Simulation

Spent the round genuinely BEING a Chinese-locale user. Findings:

**T1–T4: NO_DESKTOP_SESSION guard accepts Chinese args cleanly.** `naturo see --app 记事本 -j`, `app launch 记事本 -j`, `capture --app 记事本 -j`, `type --app 记事本 "你好世界" -j` all return the NO_DESKTOP_SESSION envelope without choking on the Chinese arg. CLI parsing layer is UTF-8-clean. ✓

**T5: INVALID_INPUT with Chinese arg routes correctly.** `naturo see --app 记事本 --depth -1 -j` → `INVALID_INPUT` with the depth-range message; the Chinese `--app` arg was accepted and processed before validation rejected `--depth -1`. ✓

**T10: dual NO_DESKTOP_SESSION messages.** `app launch 记事本` returns a DIFFERENT NO_DESKTOP_SESSION message ("Cannot launch GUI applications in a non-interactive session…") vs `see/capture` which return ("No interactive desktop in this session…"). Same code, two human-facing strings — minor, but tracked as part of the #885 epic's centralization.

**T18–T20: #893 confirmed (TC-0079 FAIL).** `wait --gone <anything> --timeout 5 -j` → success:true exit 0, wall-clock 0.25s. Also confirmed with Chinese selector `wait --gone "你好世界很长的中文文字" -j` — identical response, confirming the silent-success path predates any selector evaluation. Comment added to #893.

**T21–T24: Chinese clipboard round-trip — text correct, but JSON-escaped.** `naturo clipboard set "你好世界" -j` → `naturo clipboard get -j` → `{"text": "你好世界", "length": 4}`. Round-trip is correct (`length: 4`, parses back), but raw output uses `\uXXXX` escapes. Emoji `🎯🚀` → surrogate-pair escapes `🎯🚀`.

**T30–T31: error envelope echoes Chinese path as escapes.** `naturo type --file "C:\不存在的中文.txt" -j` → `"message": "File not found: C:\\不存在的中文.txt"`. A Chinese user reading their own error output sees `不存在的中文` instead of `不存在的中文`. → **FILED AS #894**.

**T33 — MCP capture_window with Chinese title.** Returned `INTERNAL_ERROR + NaturoCoreError: capture_window: System/COM error` with `isError: false`. This is the #881 pattern (C++ exception leak + wrong code) for a tool not previously listed in that issue. Scope-extension comment added to #881.

**R4 — `naturo app windows -j` exposed live window list with Chinese titles** in NO_DESKTOP_SESSION. Confirms #878 silent-bypass AND provides further evidence for #894: 14 windows enumerated, every Chinese title (`计算器`, `文件资源管理器`, `要恢复页面吗？`, `Windows 输入体验`) emitted as `\uXXXX` runs in the JSON bytestream.
