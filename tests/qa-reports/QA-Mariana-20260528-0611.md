## Phase 4 — Scripter/Automator Persona Findings

The Scripter mental model: **chain commands via stdout pipes, branch on exit codes, parse JSON with `jq`**. Three new contract bugs filed:

### #875 — P1 silent-failure: `dialog detect / taskbar list / tray list -j` return `success:true []` in NO_DESKTOP_SESSION

Same regression class as closed #305 / #373. These three commands lack the `_has_desktop_session` guard (verified via grep on `naturo/cli/system/_{dialog,taskbar,tray}.py` — zero matches). A Scripter loop checking `.success` would conclude "taskbar empty" instead of recognizing the desktop was unreachable.

```bash
$ naturo -j taskbar list
{"success": true, "items": [], "count": 0}   # ← lies; should be NO_DESKTOP_SESSION
$ naturo -j dialog detect
{"success": true, "dialogs": [], "count": 0}  # ← lies
$ naturo -j tray list
{"success": true, "icons": [], "count": 0}    # ← lies
```

### #876 — P2 envelope inconsistency: `selector list -j` returns `{}`, `record list -j` returns bare `{"recordings":[...]}` — no top-level `success` field

Two outliers in the otherwise-uniform `-j` envelope.

```bash
$ naturo -j selector list
{}                                    # ← no .success
$ naturo -j record list
{"recordings": [...]}                 # ← no .success, no .count
```

### #877 — P2 error envelope: `get/set -j` use `code: "UNKNOWN_ERROR"` and omit `suggested_action` on stale-snapshot ref

The failure is fully known (named in the message: *"not found in snapshot cache. Run 'naturo see' first"*) but the error envelope buries it under `UNKNOWN_ERROR` and skips `suggested_action`. Defeats `case "$(jq -r .error.code)"` dispatch.

```bash
$ naturo -j get e1
{"success": false, "error": {"code": "UNKNOWN_ERROR", "message": "Element ref 'e1' not found..."}}
                                                                                ↑ no suggested_action
```
