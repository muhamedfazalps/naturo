| MCP / AI-agent surface | ✓ 64 MCP tools | ✗ | ✗ |

**Where naturo loses ground vs. the competition**: not on capability — on **discoverability**. An evaluator on a non-interactive session (cloud VM, SSH, agent harness) has to run 6+ separate probes to answer "is my environment OK and what's missing?":

1. `naturo see -j` → desktop session?
2. `naturo desktop list -j` → pyvda? (this one *leaks the install prompt* — #869)
3. `naturo mcp tools -j` → mcp installed?
4. `naturo config show -j` → AI provider configured?
5. `naturo list screens -j` → DPI / monitor count (only works when session interactive)
6. `naturo --version` + manual PyPI check → version vs latest

That's friction an evaluator with a 5-minute attention budget will not absorb. Filed **#898** (P2 enhancement v0.3.4): `naturo doctor` — single diagnostic command + `-j` mode for agent bootstrap. Includes acceptance criteria binding it to the standard envelope contract (#876 / #884).

**Where naturo wins decisively**: cold-install survives a standard user account. pywinauto crashes on first import without admin (writing to `Program Files\Python312\Lib\site-packages\comtypes\gen\`). naturo's `NO_DESKTOP_SESSION` envelope is the calmest error of any of the three. Selector save/load, visual regression, and MCP are not even matched by pywinauto. This is the story to tell in the README — but only after the `*list -j` envelope drift is fixed, because an evaluator who pipes to `jq` will hit #876 within 2 minutes.
