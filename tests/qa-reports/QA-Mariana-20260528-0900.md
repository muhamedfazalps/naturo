## Phase 4 — Simulated User: AI Agent Builder

**Approach:** Drove `naturo mcp start` over stdio from a Python script. Sent initialize → notifications/initialized → 14 distinct `tools/call` requests across `capture_screen`, `capture_window`, `list_windows`, `list_monitors`, `see_ui_tree`, `find_element`, `focus_window`, `type_text`, `click`, `press_key`, `hotkey`, `window_resize`, `app_inspect`, and a nonexistent tool. Probed both happy-path and failure-mode contracts.

**Key findings (most-to-least surprising):**

1. **`capture_screen` returns success:true + all-black 24KB 3840×2160 PNG in NO_DESKTOP_SESSION**. Every pixel verified `(0,0,0)`. AI agents would store this as a \"screenshot\" and operate on it. **#868 confirmed** (P0 silent-failure already filed).
2. **`see_ui_tree` returns a real UIA tree** with valid bounds matching the actual claude window, even though CLI `naturo see` refuses with NO_DESKTOP_SESSION. The CLI couples \"capture screenshot\" + \"enumerate tree\"; the MCP tool only does the latter, so the guard is bypassed by accident. UIA enumeration genuinely works without graphics, but the inconsistency is confusing. → #878 scope-expansion comment.
3. **`focus_window {title: claude}` returns `{success:true, action:'focus'}`** — unverifiable from this session whether SetForegroundWindow actually moved focus. → #878 scope-expansion comment.
4. **MCP error codes**: `capture_window {nonexistent_title}` → `INTERNAL_ERROR / NaturoCoreError: capture_window: System/COM error`. Should be `WINDOW_NOT_FOUND`. `type_text` / `click` in NO_DESKTOP → `INTERNAL_ERROR / NaturoCoreError: key_type/mouse_move: System/COM error`. Should be `NO_DESKTOP_SESSION`. **Filed #881 (P1)**.
5. **MCP `isError` flag**: five different shapes across six failure modes. `press_key` / `hotkey` in NO_DESKTOP → `isError:true`, no payload. `window_resize` with type error → `isError:true`, raw Pydantic text. `find_element` no-match / `capture_window` nonexistent / `click {}` → `isError:false`, JSON payload with `success:false`. Agents must check both fields. **Filed #882 (P2)**.
6. **MCP `serverInfo.version` = `1.26.0`** (MCP SDK version), not `0.3.1` (naturo). **#873 confirmed**.
7. **MCP `nonexistent_tool` → `isError:true, content:'Unknown tool: nonexistent_tool'`**. Plain string, not JSON — consistent with the inconsistent `isError` mapping in #882.
8. **`window_resize {width:'not_a_number', ...}` → raw Pydantic error string leaks `errors.pydantic.dev/2.12/v/int_parsing` URL to agent**. **#844 confirmed**.

The MCP surface is the **primary AI-agent entrypoint**. Three new contract-level findings here (#881, #882, plus the #878 scope expansion) suggest the MCP layer needs a unified response-envelope helper, similar to what the CLI uses for `-j` mode.
